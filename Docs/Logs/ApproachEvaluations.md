# アプローチ検証結果と評価 (Approach Evaluations)

このドキュメントは、各アプローチを「単体」または「組み合わせ」で導入した際の検証結果をまとめ、最終的な「導入可否（採用/不採用）」を判定するための記録です。

---

## 評価基準 (Evaluation Criteria)
* **パフォーマンス向上**: FPSの改善、サーバーCPUの低下、通信帯域の削減。
* **実装コスト/複雑さ**: 導入によるコードの複雑化や保守性の低下リスク。
* **組み合わせの相性**: 他のアプローチと同時に適用した際、不具合（競合）が生じないか、または相乗効果があるか。

---

## 検証ログ (Evaluation Logs)

### [検証中] 標準物理演算（Rigidbody） vs 独自XZ物理演算
* **検証日時**: 2026-05-02
* **検証環境**: WebGL (Client) + サーバー (Host/Dedicated)
* **結果 (単体の良し悪し)**: 
  * [標準物理] 
    * [Good] 950個という過剰な数でなければ、Photonの `NetworkRigidbody3D` とUnity標準物理エンジンで全く問題なく、非常に軽く動作する。
    * [Bad] `MeshCollider` を動かすには `Convex`（凸包）化が必須だが、これを行うと机の下の空間などが埋まってしまい、形状が雑（大雑把）になってしまう。
    * [Bad] 正確な形状を作るには、天板と4本の足にそれぞれ `BoxCollider` を手作業で配置する（Compound Collider）必要があり、配置数が多すぎると負荷が跳ね上がる。
  * [独自物理]
* **最終判定**: 🟡 保留 (Pending)
* **特記事項**: 950個のオブジェクトを同時に揺らした際の処理負荷（FPS）と、同期の安定性を比較する。
