研究室のPCでクローンしてWebGLビルドしてビルドしたディレクトリで上記コマンドを実行すると一時URLを作成できる。このURLを共有することで外部のPCからアクセス可能。

Photonをサーバーモードで機能させる
物理演算はサーバーにさせてクライアントが見るオブジェクトの位置や角度を完全に一致させ、処理速度をクライアント依存にさせない。
Rigidbodyの衝突計算などをWebGL側では一切行わないようにし、Photon Fusionのサーバーに全て委ねた。
client FPSと　Server FPSを画面に表示
どちらも問題ないがかくつく原因を究明するため受信パケット数、送信パケット数、RTT(Round Trip Time。いわゆるPing値サーバーに送ってから帰ってくるまでの時間)、送受信帯域(InPackets, OutPackets, RTT, InBandwidth, OutBandwidth)を表示
かくついていないとき→受信パケット数低、送信パケット数高、RTT低、受信帯域低、送信帯域高
かくついているときはその逆→Photonサーバーの処理負荷によって引き起こされていると判明。

地震などのすべてのオブジェクトをAwake状態にする動作はアニメーションベイクを用いて物理演算を用いない方向にして、ワールドを個別に自由に歩く際は物理演算を用いるように変更しようと考えている。
