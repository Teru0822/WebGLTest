# Simulation Design & Logic

このドキュメントでは、地震避難訓練シミュレーションのゲームロジックおよびシーン設計について定義します。

## 1. シーン構成
- **環境オブジェクト**: 床、壁などは静的オブジェクトとして配置し、ネットワーク同期は行いません（全クライアント共通のシーンデータとして保持）。
- **動的オブジェクト (物理演算対象)**:
  - 机、椅子、本棚などの家具類。
  - これらはプレハブ化し、`NetworkObject` と `NetworkRigidbody` をアタッチしてサーバー側でSpawnさせます。
  - シーン上に初期配置されている場合も、サーバー起動時にサーバーがNetworkObjectとして状態を取得・管理します。

## 2. 地震の発生メカニズム (EarthquakeManager)
- サーバー上で動作する管理スクリプトが地震の発生を制御します。
- **揺れのロジック**:
  - 特定のタイミング（ランダムまたは一定間隔）で、シーン内の特定のオブジェクト（または全て）に対して、Rigidbodyの `AddForce` や `AddTorque` を用いてランダムな方向・強さの力を加えます。
  - 地震の「強度（Intensity）」パラメータを用意し、それに応じて加える力の大きさを変動させます。
- **クライアントへの通知**:
  - 地震が発生しているという状態（UI表示やカメラ揺れ用）は、Networkedプロパティ（例: `[Networked] public bool IsQuaking { get; set; }`）を用いて全クライアントに同期します。

## 3. プレイヤー（クライアント）の挙動
- クライアントはログイン後、自身のアバター（プレイヤーオブジェクト）をSpawnします。
- プレイヤーは自由に移動でき、地震発生時は「机の下に隠れる」などのアクションを行えるよう、将来的にインタラクション機能を追加します。
- プレイヤーの物理挙動（移動）はNetworkCharacterController等を用いて実装し、サーバーで検証（Server-Auth）を行います。
