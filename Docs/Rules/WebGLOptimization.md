# WebGL パフォーマンス最適化ガイド (WebGL Optimization Guide)

WebGLビルドにおいて、高いFPS（60FPS）と安定した動作を実現するための必須設定とスクリプトのまとめです。
ブラウザ特有の制限やシングルスレッドのボトルネックを回避するために、どの環境でも必ずこの設定を行ってください。

## 1. ブラウザの設定 (必須)
*   **ハードウェア アクセラレーションの有効化**:
    *   Chromeの場合、`chrome://settings/system` を開き「グラフィック アクセラレーションが使用可能な場合は使用する」をオンにします。
    *   これがオフの場合、CPUによるソフトウェアレンダリングとなり、FPSが一桁まで落ち込みます。

## 2. 描画負荷の削減 (GPU Instancing)
WebGLのCPU（シングルスレッド）がパンクする最大の原因は「ドローコール（描画命令）の多さ」です。
*   **GPU Instancingの有効化**:
    *   同じ形状のオブジェクト（机、椅子など）を大量に配置する場合、必ずマテリアルの `Enable GPU Instancing` にチェックを入れてください。
    *   これにより、500個の机を描画する命令が500回から1回に減少し、劇的に軽くなります。

## 3. Unityビルド・品質設定の最適化
以下の設定を必ず適用してください。エディタ上部のメニュー `WebGLTest` ＞ `🚀 WebGLのパフォーマンスを最適化する (16FPS対策)` をクリックすることで自動適用されます。
*   **Quality Settings**:
    *   `vSyncCount = 0` (WebGL環境でのフレームレート乱高下を防ぐ)
    *   `Shadow Cascades = 1` または 0 (影の処理を極力軽くする)
    *   `Anti Aliasing = 0` (ポストプロセスの負荷削減)
*   **Player Settings (WebGL)**:
    *   `Exception Support = None` (例外監視によるC#→JS変換の甚大なオーバーヘッドを完全にカットする)

## 4. クライアント側の物理演算オーバーヘッドのカット
Photon Fusionを用いた Server/Client 構成において、クライアント側ではオブジェクトの座標がサーバーから補間（Interpolation）されて動きます。
*   **`ClientColliderDisabler.cs` のアタッチ**:
    *   クライアント側で動くオブジェクトに `MeshCollider` が付いていると、座標更新のたびにUnity内部の物理エンジン（PhysX）が再計算を行い、致命的な処理落ち（16FPS問題）が発生します。
    *   すべての物理オブジェクトのプレハブに `ClientColliderDisabler` をアタッチし、クライアント側でのみColliderを無効化してください。

## 5. 高解像度モニター(4K等)での描画崩壊対策
*   **`TargetFPS.cs` による解像度制限**:
    *   WebGLはブラウザのサイズに合わせて描画ピクセル数が際限なく大きくなります。4Kモニターで全画面表示するとグラフィック処理が破綻します。
    *   `TargetFPS.cs` をシーンに配置することで、内部のレンダリング解像度の上限をフルHD（1920x1080）に制限し、どんなモニターでも安定した負荷を維持します。
