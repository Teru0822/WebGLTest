# Vibe Coding Workflow

このドキュメントは、本プロジェクトにおけるAIエージェント（Claude、Gemini）との開発ワークフローを定義します。

## 1. エージェントの役割
- **Claude**: メインのエージェントとして使用。複雑なアーキテクチャ設計や、C#スクリプトの大量のコーディング、Photon Fusionの仕様に則った実装を主に担当。
- **Gemini**: サブエージェントとして使用。問題解決時の壁打ち、特定のエラーやツールのデバッグ、実装計画のレビュー等に使用。

## 2. 開発の流れ
1. **要件定義**: ユーザーはプロンプトで「実現したいこと」を伝えます。
2. **ルール確認**: エージェントは必ず `AGENTS.md` を起点に関連ドキュメントを確認します。
3. **Planの作成**: 複雑な機能を追加する場合は、エージェントは `implementation_plan.md` を作成し、ユーザーの承認（Approve）を得てから実装に進みます。
4. **実装とタスク管理**: 実装中は `task.md` を使用して進行状況を可視化します。
5. **完了報告**: 実装が完了したら `walkthrough.md` でユーザーに変更内容と確認手順を報告します。

## 3. コード記述ルール
- Photon Fusionのクラス（NetworkBehaviour等）を使用する際は、名前空間 `Fusion` を明記してください。
- 物理演算はサーバー側で行うため、Rigidbodyの制御コードは `if (HasStateAuthority)` などの権限チェックで保護してください。
- メソッドやクラスの役割を明確にするため、XMLドキュメントコメント（`/// <summary>`）を適切に付与してください。
